Agencia de Exploradores

Esta actividad NO es un simulacro de examen. Su extensión supera lo que se solicitará en el examen de la convocatoria ordinaria. Además, en este modelo se facilita información para la autocorrección de la prueba (rubrica detallada).

Se pretende que sirva de preparación para la prueba real ya que el formato de enunciado será similar exceptuando los detalles de la rúbrica y los elementos válidos e inválidos esperados en los sets de datos.

Contexto​

La agencia Rutas Salvajes organiza expediciones guiadas por distintos entornos naturales. Para mejorar su gestión interna, necesita una aplicación en Python que permita administrar exploradores, expediciones y reservas.

Se deberá implementar el sistema siguiendo los principios de la Programación Orientada a Objetos.

No se permite acceder directamente a los atributos de las clases desde fuera de ellas. Se deberán utilizar propiedades, getters, setters o métodos específicos cuando sea necesario.

Parte 1: Gestión de exploradores​

Modifica el fichero explorador.py para crear la clase Explorador.

Clase Explorador​

Atributos de instancia​

id_explorador
nombre
nivel
presupuesto
  • id_explorador: cadena de texto.
  • nombre: cadena de texto.
  • nivel: cadena de texto. Puede ser "principiante", "intermedio" o "experto".
  • presupuesto: número real.

Restricciones​

  1. El presupuesto no puede ser negativo.
  2. El nivel solo podrá tener uno de estos valores:
principiante
intermedio
experto
  1. Si se intenta asignar un presupuesto negativo, se lanzará un ValueError.
  2. Si se intenta asignar un nivel no válido, se lanzará un ValueError.

Método mágico​

Implementa el método mágico necesario para que, al imprimir un explorador, se muestre con el siguiente formato:

<nombre> (<nivel>) - Presupuesto: <presupuesto> €

Ejemplo:

Laura (intermedio) - Presupuesto: 450.0 €

Parte 2: Expediciones​

Modifica el fichero expediciones.py.

Se deberá crear una clase base Expedicion y dos clases hijas:

ExpedicionMontana
ExpedicionSelva

Clase Expedicion​

Atributo compartido​

Todas las expediciones tendrán un atributo compartido llamado:

empresa

Su valor por defecto será:

Rutas Salvajes

Atributos de instancia​

id_expedicion
nombre
dificultad
duracion_dias
precio_base
plazas_totales
plazas_ocupadas
  • id_expedicion: cadena de texto.
  • nombre: cadena de texto.
  • dificultad: "baja", "media" o "alta".
  • duracion_dias: número entero.
  • precio_base: número real.
  • plazas_totales: número entero.
  • plazas_ocupadas: número entero. Por defecto será 0.

Restricciones​

  1. precio_base no puede ser negativo.
  2. duracion_dias debe ser mayor que 0.
  3. plazas_totales debe ser mayor que 0.
  4. plazas_ocupadas no puede ser negativa.
  5. plazas_ocupadas no puede ser mayor que plazas_totales.
  6. dificultad solo podrá ser "baja", "media" o "alta".

Si alguna de estas restricciones no se cumple, se lanzará un ValueError.

Métodos de la clase Expedicion​

  • calcular_precio_final()

Debe devolver el precio final de la expedición.

El cálculo será:

precio_final = precio_base * multiplicador_dificultad

Los multiplicadores serán:

baja → 1
media → 1.25
alta → 1.5
  • hay_plazas_disponibles()

Debe devolver True si quedan plazas disponibles y False en caso contrario.

  • reservar_plaza()

Debe aumentar en 1 el número de plazas ocupadas.

Si no hay plazas disponibles, deberá lanzar un ValueError.

  • cancelar_plaza()

Debe reducir en 1 el número de plazas ocupadas.

Si no hay plazas ocupadas, deberá lanzar un ValueError.

  • realizar_expedicion()

Método polimórfico.

En la clase base debe lanzar una excepción NotImplementedError.

Clases hijas​

Clase ExpedicionMontana​

Hereda de Expedicion.

Atributo específico

altura_maxima

Será un número entero que representa la altura máxima de la ruta en metros.

Sobrescritura de realizar_expedicion()

Debe devolver el siguiente mensaje:

Ascendiendo hasta <altura_maxima> metros en la expedición <nombre>.

Modificación del precio

Las expediciones de montaña tendrán un suplemento fijo de 40 € sobre el precio final.

Por tanto, en esta clase deberá sobrescribirse el método calcular_precio_final().

Clase ExpedicionSelva​

Hereda de Expedicion.

Atributo específico

riesgo_fauna

Será una cadena de texto que podrá tener los valores:

bajo
medio
alto

Si el valor no es válido, se lanzará un ValueError.

Sobrescritura de realizar_expedicion()

Debe devolver el siguiente mensaje:

Explorando la selva con riesgo de fauna <riesgo_fauna> en la expedición <nombre>.

Modificación del precio

Las expediciones de selva con riesgo de fauna "alto" tendrán un suplemento de 60 € sobre el precio final.

Las expediciones de selva con riesgo "bajo" o "medio" no tendrán suplemento.

Parte 3: Objeto Agencia​

Crea la clase Agencia en el fichero agencia.py.

La agencia deberá gestionar exploradores, expediciones y reservas.

Atributos​

exploradores
expediciones
reservas
  • exploradores: diccionario donde la clave será el ID del explorador y el valor será un objeto Explorador.
  • expediciones: diccionario donde la clave será el ID de la expedición y el valor será un objeto ExpedicionMontana o ExpedicionSelva.
  • reservas: lista de diccionarios con la información de las reservas realizadas.

No se podrán modificar directamente estos atributos desde fuera de la clase.

Parte 4: Métodos de la agencia​

La clase Agencia deberá contener los siguientes métodos.

agregar_explorador(explorador)​

Recibe un objeto Explorador.

Debe añadirlo al diccionario de exploradores.

Si ya existe un explorador con el mismo ID, se lanzará un ValueError.

agregar_expedicion(expedicion)​

Recibe un objeto de tipo Expedicion.

Debe añadirlo al diccionario de expediciones.

Si ya existe una expedición con el mismo ID, se lanzará un ValueError.

buscar_explorador(id_explorador)​

Recibe el ID de un explorador.

Debe devolver el objeto Explorador correspondiente.

Si no existe, se lanzará un ValueError.

buscar_expedicion(id_expedicion)​

Recibe el ID de una expedición.

Debe devolver el objeto Expedicion correspondiente.

Si no existe, se lanzará un ValueError.

realizar_reserva(id_explorador, id_expedicion)​

Recibe el ID de un explorador y el ID de una expedición.

Debe realizar una reserva si se cumplen todas estas condiciones:

  1. El explorador existe.
  2. La expedición existe.
  3. La expedición tiene plazas disponibles.
  4. El explorador tiene presupuesto suficiente para pagar la expedición.

Si se realiza la reserva:

  • Se ocupará una plaza en la expedición.
  • Se restará el precio final de la expedición al presupuesto del explorador.
  • Se añadirá una reserva a la lista de reservas.

La reserva se guardará como un diccionario con esta estructura:

{
    "id_explorador": "EX001",
    "id_expedicion": "EP001",
    "precio_pagado": 275.0
}

Si no se puede realizar la reserva, se lanzará un ValueError.

cancelar_reserva(id_explorador, id_expedicion)​

Debe cancelar una reserva existente.

Si existe la reserva:

  • Se libera una plaza de la expedición.
  • Se devuelve el importe pagado al presupuesto del explorador.
  • Se elimina la reserva de la lista.

Si la reserva no existe, se lanzará un ValueError.

expedicion_mas_cara()​

Debe devolver el objeto expedición con mayor precio final.

expediciones_disponibles()​

Debe devolver una lista con las expediciones que tienen plazas disponibles.

recaudacion_total()​

Debe devolver la suma de todos los precios pagados en las reservas realizadas.

tipo_mas_rentable()​

Debe devolver qué tipo de expedición ha generado mayor recaudación.

Devolverá uno de estos textos:

montana
selva
empate
sin reservas
