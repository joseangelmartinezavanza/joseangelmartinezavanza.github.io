5.4 Funciones de orden superior

En Python, las funciones son ciudadanos de primera clase (first-class objects), lo que significa que se pueden almacenar en variables, pasar como argumentos y devolver como resultado de otras funciones. Este comportamiento permite crear las llamadas funciones de orden superior, es decir, funciones que reciben otras funciones como par√°metro o las devuelven como resultado.

Este concepto procede de la programaci√≥n funcional y ofrece una gran flexibilidad a la hora de estructurar el c√≥digo.

1Ô∏è‚É£ Qu√© es una funci√≥n de orden superior‚Äã

Una funci√≥n de orden superior es aquella que:

  1. Recibe una funci√≥n como argumento, o
  2. Devuelve una funci√≥n como resultado, o ambas cosas a la vez.

Gracias a este mecanismo, Python permite construir estructuras m√°s expresivas y reutilizables.

Ejemplo de funci√≥n que recibe otra funci√≥n

    """Aplica la funci√≥n recibida al valor dado."""
    return funcion(valor)

    return x ** 2

en este ejemplo
  • La funci√≥n aplicar_operacion() recibe como argumento otra funci√≥n (cuadrado) y un valor (5).
  • Dentro de aplicar_operacion() se ejecuta funcion(valor), que en este caso equivale a cuadrado(5), devolviendo 25.
  • De este modo, una funci√≥n puede actuar sobre otras funciones sin conocer su implementaci√≥n concreta.

2Ô∏è‚É£ Funciones como objetos de primera clase‚Äã

En Python, las funciones son tratadas como cualquier otro tipo de dato. Esto significa que se pueden:

  • Asignar a variables,
  • Pasar como par√°metros,
  • Guardar en estructuras de datos,

  • Esta t√©cnica resulta √∫til para aplicar un conjunto de operaciones de forma din√°mica.

3Ô∏è‚É£ Funciones que devuelven otras funciones‚Äã

Una funci√≥n de orden superior tambi√©n puede devolver otra funci√≥n. Esto se utiliza, por ejemplo, para crear funciones personalizadas o configuradas din√°micamente.

4Ô∏è‚É£ Funciones de orden superior incorporadas‚Äã

Python incluye varias funciones de orden superior muy utilizadas que ya hemos presentado en el punto anterior: map(), filter() y reduce(). Estas funciones toman otra funci√≥n como argumento para aplicar transformaciones o filtrados sobre colecciones de datos.

üü© map()‚Äã

üü¶ filter()‚Äã

filter(funcion, iterable) recorre una colecci√≥n y conserva solo los elementos que hacen que la funci√≥n devuelva True.

5Ô∏è‚É£ Creaci√≥n de funciones personalizadas con closures‚Äã

Los closures son funciones internas que conservan el estado de las variables de la funci√≥n exterior incluso despu√©s de que esta haya terminado. Se utilizan para crear funciones configuradas que recuerdan un valor o contexto determinado.

        n += 1

    return incrementar

print(c())  # 3
en este ejemplo

  • Cada vez que se llama a c(), el valor interno de n se incrementa, manteni√©ndose entre ejecuciones.

Esto demuestra que las funciones pueden retener un ‚Äúestado interno‚Äù sin usar variables globales.

8Ô∏è‚É£ Decoradores‚Äã

9Ô∏è‚É£ Generadores‚Äã

Los generadores son un tipo especial de funci√≥n que permite producir una secuencia de valores de forma progresiva, en lugar de devolverlos todos de una vez.   • Cuando el bucle for solicita el siguiente valor, la ejecuci√≥n se reanuda desde la l√≠nea siguiente al yield.
  • El proceso contin√∫a hasta que se alcanza el final de la funci√≥n.
