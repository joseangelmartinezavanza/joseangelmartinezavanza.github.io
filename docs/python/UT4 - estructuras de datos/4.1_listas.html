4.1 Listas (list)

1Ô∏è‚É£ ¬øQu√© es una lista y para qu√© sirve?‚Äã

En Python, una lista es un tipo de dato complejo que permite almacenar varios valores en una sola variable. A diferencia de los tipos simples (como un n√∫mero o una cadena), una lista puede contener una colecci√≥n ordenada de elementos, y cada uno de ellos puede ser de un tipo diferente.

Las caracter√≠sticas principales de una lista son:

  • Ordenadas: mantienen el orden en que se introducen los elementos.
  • Mutables: se pueden modificar (a√±adir, eliminar o cambiar elementos).
  • Heterog√©neas: pueden incluir datos de distintos tipos (n√∫meros, cadenas, booleanos, etc.).
  • Permiten duplicados: se pueden repetir valores sin problema.
Ejemplo de declaraci√≥n de una lista

en este ejemplo

En este caso, la lista mi_lista contiene cuatro elementos de distintos tipos. Python no exige que todos los elementos sean del mismo tipo, lo que hace que las listas sean muy flexibles.

8Ô∏è‚É£ Copia de listas‚Äã

Cuando se trabaja con colecciones (listas, tuplas o diccionarios), es importante entender c√≥mo funcionan las copias de los objetos. En Python, asignar una variable a otra no crea una copia, sino que ambas apuntan al mismo objeto en memoria. Para evitar modificar datos por error, Python ofrece dos formas de copiar: copias superficiales (copy) y copias profundas (deepcopy).

üü© Asignaci√≥n simple (sin copia)‚Äã

Ejemplo de asignaci√≥n simple

b = a        # b no es una copia, es la misma lista
b.append(4)
print("a:", a)
print("b:", b)

# a: [1, 2, 3, 4]
# b: [1, 2, 3, 4]
en este ejemplo

Tanto a como b apuntan al mismo objeto en memoria. Modificar uno afecta al otro.

üü¶ Copia superficial (copy())‚Äã

Una copia superficial crea un nuevo contenedor, pero no copia los objetos internos (solo sus referencias). Esto significa que si la lista contiene otras listas u objetos, esos elementos seguir√°n comparti√©ndose.

Ejemplo de copia superficial con listas anidadas

copia = original.copy()

copia[0][0] = 99

print("Original:", original)
print("Copia:", copia)

# Original: [[99, 2], [3, 4]]
# Copia: [[99, 2], [3, 4]]
en este ejemplo

Aunque copia es una lista distinta, las sublistas internas son las mismas. Por eso, cambiar un elemento interno tambi√©n afecta al original.

Atenci√≥n:

Si la lista contiene otras listas dentro, la copia no ser√° totalmente independiente (para eso se necesita una copia profunda usando copy.deepcopy() del m√≥dulo copy).

üü® Copia profunda (deepcopy())‚Äã

Una copia profunda crea un nuevo contenedor y tambi√©n copias independientes de todos los objetos internos, de forma recursiva. Esto garantiza que los cambios en la copia no afecten al original.

Ejemplo de copia profunda con listas anidadas
import copy

original = [[1, 2], [3, 4]]
copia_profunda = copy.deepcopy(original)

copia_profunda[0][0] = 99

print("Original:", original)
print("Copia profunda:", copia_profunda)

# Original: [[1, 2], [3, 4]]
# Copia profunda: [[99, 2], [3, 4]]

en este ejemplo

deepcopy() ha creado copias completamente nuevas de las sublistas internas, por lo que los cambios en una no afectan a la otra.

9Ô∏è‚É£ Recorrido de listas‚Äã

Una de las operaciones m√°s comunes que se realizan con listas es recorrer sus elementos uno a uno, lo que se denomina iterar. Iterar significa ejecutar una acci√≥n repetida para cada elemento de la lista: mostrarlo, modificarlo, aplicar una operaci√≥n o comprobar una condici√≥n.

Python ofrece distintas formas de recorrer listas de manera clara y eficiente. En este apartado se estudian todas ellas.

üü© Recorrido b√°sico con el bucle for‚Äã

La forma m√°s directa de recorrer una lista es utilizando un bucle for, que permite acceder a cada elemento directamente sin preocuparse por su posici√≥n.

üëâ Este tipo de recorrido es ideal cuando solo necesitas el valor de cada elemento y no te importa su posici√≥n en la lista.

En algunas situaciones puede interesar acceder al √≠ndice (posici√≥n) de cada elemento adem√°s de su valor. Para ello, se puede usar la funci√≥n range() junto con len(), que devuelve la longitud de la lista.

  • range(len(numeros)) genera la secuencia [0, 1, 2, 3].
  • En cada iteraci√≥n, la variable i representa el √≠ndice actual, y numeros[i] el valor correspondiente.

üëâ Este m√©todo es √∫til cuando se necesita modificar los valores de la lista mientras se recorren, ya que permite acceder a ellos mediante su posici√≥n.

Python ofrece una forma m√°s c√≥moda y legible de obtener √≠ndices y valores a la vez mediante la funci√≥n integrada enumerate().

En ocasiones se necesitan recorrer varias listas al mismo tiempo, tomando un elemento de cada una en cada vuelta del bucle. Para esto se utiliza la funci√≥n zip(), que ‚Äúempareja‚Äù los elementos de las listas seg√∫n su posici√≥n.

üîü Ordenaci√≥n de listas‚Äã

En Python, las listas pueden reorganizarse gracias a m√©todos y funciones integradas como sort(), sorted() y reverse(). Estos permiten controlar el orden de los elementos y seleccionar solo aquellos que cumplan ciertas condiciones.

üü© sort(key=None, reverse=False)‚Äã

El m√©todo sort() ordena los elementos en orden ascendente por defecto. Modifica la lista original y no devuelve una nueva lista.

Ejemplo b√°sico de sort()
